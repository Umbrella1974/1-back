scheduler 状态机：

WAIT_OPEN_ZONE:
  等待 pinch zone 进入 open_zone。
  进入 open_zone 后，触发 plan.events[0]，它必须是 contact。
  contact 发出后进入 WAIT_CLOSED_ZONE。

WAIT_CLOSED_ZONE:
  等待 pinch zone 进入 closed_zone。
  进入 closed_zone 后，开始执行 plan.events[1:]。
  也就是 slip/left/right/.../release。
  进入 RUN_PLAN_EVENTS。

RUN_PLAN_EVENTS:
  按 plan events 顺序输出事件。
  after_previous 的 gap_ms 要生效。
  每个 event 的 duration_ms 要用于计算当前事件结束时间。
  release 必须作为最后一个事件正常输出。
  全部完成后进入 REFRACTORY。

REFRACTORY:
  在 refractory_ms 内不允许新 trial。
  refractory 结束后回到 WAIT_OPEN_ZONE。

要求：
1. contact 只在 open_zone 触发。
2. contact 后不要立刻自动发 slip/left/right/release，必须等进入 closed_zone。
3. closed_zone 后按 plan config 顺序发中间事件和 release。
4. release 不要由 scheduler 自动追加；必须来自 plan config。plan config 已经校验最后一个是 release。

需要修正 haptic_trial_scheduler 的时序需求。

不要写成“进入 open_zone 立刻触发 contact”。
contact 需要在进入 open_zone 后，经过一个伪随机延迟再触发。

也不要写成“closed_zone 后每个事件使用固定 gap_ms”。
进入 closed_zone 后，按 haptic plan config 的顺序发送 slip/left/right/release，但每个事件的 onset 间隔需要从 config 中的随机范围抽取。
每个事件自己的 duration_ms 是固定值，由 config 指定。

具体状态机应改成：

WAIT_OPEN_ZONE:
  等待 pinch zone 进入 open_zone。

PENDING_CONTACT:
  进入 open_zone 后，抽取 contact_onset_delay_ms。
  planned_contact_onset_ms = enter_open_zone_ms + sampled_delay_ms。
  如果等待期间 pinch 离开 open_zone，则取消 pending contact，回到 WAIT_OPEN_ZONE。
  如果到达 planned_contact_onset_ms 且仍在 open_zone，则发送 contact。
  contact 发送后进入 WAIT_CLOSED_ZONE。

WAIT_CLOSED_ZONE:
  等待 pinch zone 进入 closed_zone。
  进入 closed_zone 后，准备执行 plan 中 contact 后面的事件。
  注意：contact 不应在这里重复发送。

PENDING_PLAN_EVENT:
  对下一个 plan event 抽取随机 onset gap。
  planned_event_onset_ms = previous_event_end_ms + sampled_gap_ms。
  如果当前事件需要在 closed_zone 触发，则等待期间必须仍在 closed_zone；如果离开 closed_zone，按 config 决定取消、暂停或继续。
  到达 planned_event_onset_ms 后，发送该事件。
  发送后记录 event duration_ms，并计算 event_end_ms。
  如果还有下一个事件，继续进入 PENDING_PLAN_EVENT。
  如果当前事件是 release 且是最后一个事件，进入 REFRACTORY。

REFRACTORY:
  一个 haptic trial 完成后，在 refractory_ms 内不允许新 trial。
  refractory 结束后回到 WAIT_OPEN_ZONE。

contact 使用 onset_delay_ms；如果 contact 没写，则使用 timing.contact_onset_delay_ms 默认值。

随机性要求：
- 使用 plan config 的 random_seed 创建 random.Random(seed)。
- 这样同一个 plan 和 seed 可以复现实验时序。
- 每次 haptic trial 都要重新抽取 delay/gap，而不是全 session 只抽一次。
- 日志里必须记录每个 event 实际抽到的 sampled_delay_ms 或 sampled_gap_ms。

digit onset guard：
- contact、release 必须应用 digit onset guard。
- slip、left、right 也可以应用，但优先级低。
- 第一版可以对所有 event onset 统一应用 guard，但日志里要能看出 original_planned_onset_ms 和 adjusted_onset_ms。
- guard 只调整 event 开始时刻，不调整 duration_ms。

测试需要相应修改：
