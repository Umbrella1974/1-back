当前不要改 MANUS TCP 8888 接口，不要改 scheduler，不要改 nback 逻辑。

现在只实现真实 haptic TCP sender。

现状：
1. SimpleHapticSenderConfig 仍然 disabled_mode=True。
2. disabled_mode=False 会 NotImplementedError。
3. run_pinch_haptic_1back.py 创建 SimpleHapticSenderConfig 时写死 disabled_mode=True。
4. dualtask_config.yaml 没有 vibration_tcp / matrix_tcp 配置。
5. vendor_exp2_abc 里已经有：
   - haptic_tcp_worker.py / MatrixTcpWorker
   - vibration_tcp_worker.py / VibrationTcpLineWorker
   - matrix_haptic_protocol.py
   - vibration_haptic_protocol.py

目标：
让 SimpleHapticSender 支持真实 TCP，但保留 disabled 模式。

修改要求（参数严格按照exp2复制过来的文件）：

1. dualtask_config.yaml 增加：

vibration_tcp:
  enabled: false
  required: false
  host: 192.168.x.y
  port: 12346
  connect_timeout_s: 2.0
  send_timeout_s: 0.2

matrix_tcp:
  enabled: false
  required: false
  host: 192.168.x.y
  port: 12345
  connect_timeout_s: 2.0
  send_timeout_s: 0.2

2. SimpleHapticSenderConfig 增加：
- disabled_mode
- vibration_tcp_enabled
- vibration_required
- vibration_host
- vibration_port
- matrix_tcp_enabled
- matrix_required
- matrix_host
- matrix_port
- timeouts

3. run_pinch_haptic_1back.py 不要写死 disabled_mode=True。
应该从 config 读取：
- haptic.vibration_enabled
- haptic.matrix_enabled
- vibration_tcp.enabled
- matrix_tcp.enabled

建议语义：
- 如果 haptic.vibration_enabled=false，则 vibration 不连接。
- 如果 haptic.vibration_enabled=true 且 vibration_tcp.enabled=true，则启动 VibrationTcpLineWorker。
- 如果 haptic.matrix_enabled=false，则 matrix 不连接。
- 如果 haptic.matrix_enabled=true 且 matrix_tcp.enabled=true，则启动 MatrixTcpWorker。
- required=false 时连接失败只 warning，继续实验并记录 send_status。
- required=true 时连接失败报错并停止实验。

4. SimpleHapticSender 初始化时：
- disabled 模式不连接任何 TCP。
- vibration enabled 时启动 VibrationTcpLineWorker。
- matrix enabled 时启动 MatrixTcpWorker。
- 程序结束时 stop/close worker，避免后台线程残留。

5. event 发送映射：
- modality=vibration:
  使用 command_id 或 command_label 转成 vibration payload。
  contact -> command_id 1
  release -> command_id 2
  slip -> command_id 3
- modality=matrix:
  使用 channel_list 通过 matrix_haptic_protocol 打包 payload。
  left/right 不要写死 channel，使用 plan config 的 channel_list。

6. 发送日志：
每个 event 写：
- tcp_enabled
- tcp_queued
- tcp_success
- send_status
- note

命令行打印：
[HAPTIC] event=contact modality=vibration command_id=1 tcp=queued
[HAPTIC] event=left modality=matrix channels=[1,2,3] tcp=queued

7. 测试：
新增 fake socket / fake worker 测试，不要依赖真实 ESP32。
- vibration enabled 时会 queue command_id
- matrix enabled 时会 queue channel_list payload
- disabled 时不连接
- required=false 连接失败不崩
- required=true 连接失败报错
- enabled=false 的通道不连接
- stop/close 会被调用
