当前 run_pinch_haptic_1back.py 已经能完成真实 MANUS + 1-back + haptic disabled 双任务运行。

最新输出结果：

* calibration_passed=true
* distance_range 正常
* total_nback_trials=50
* total_nback_responses=46
* total_pinch_samples=7297
* total_haptic_events=28
* warnings/errors 为空

但当前实验语义需要修改。

核心问题：
当前程序在 release 后还会继续等待下一轮 open_zone，因此会触发多轮 haptic trial。
而我的实验需求是：
一次 session 中只需要一轮 haptic episode。
当 release 被触发后，这一轮 haptic 和 1-back 都应该结束。
不要 release 后又张开手进入新一轮。
不要让 1-back 固定跑满 50 trials 后把 haptic sequence 截断。



目标 1：release 结束整个双任务 session

新增 config：

session:
end_policy: stop_on_haptic_release
allow_multiple_haptic_trials: false
finish_active_haptic_before_exit: true

语义：

1. allow_multiple_haptic_trials=false 时，整个正式阶段只允许一轮 haptic trial。
2. 一旦 release event 被 emit，就设置 session_should_end=true。
3. release emit 后：

   * haptic scheduler 停止；
   * 1-back 主循环结束；
   * 写出 nback_events.csv、pinch_timeseries.csv、haptic_events.csv、summary.json；
   * summary.json 里记录 end_reason="haptic_release"。
4. 如果 1-back 自己先到达原本 trial 数上限，但 haptic sequence 正在 active：

   * 如果 finish_active_haptic_before_exit=true，则继续主循环直到 release 完成，再结束；
   * 如果 finish_active_haptic_before_exit=false，则允许结束，但 summary 里必须 warning：haptic_sequence_interrupted。
5. 如果 session duration 到达但 haptic sequence active，也按同样策略处理。
6. 不允许出现 session 结束时 haptic trial 只有 contact/slip/left 但没有 release，除非 config 明确允许 interrupt。

目标 2：haptic plan 支持任意数量的中间触觉信号

当前 plan 只适合 contact -> slip -> left -> right -> release。
需要改成：

* contact 仍必须是第一个；
* release 仍必须是最后一个；
* 中间事件数量不限；
* 中间事件可以是 3 个、6 个、9 个，或者更多；
* 中间事件顺序完全由 haptic_plan_config.yaml 的 events 列表决定；
* 不要在 scheduler 里写死 slip/left/right 数量；
* haptic_trial_scheduler 必须按 events 列表从头到尾执行。

事件合法性：

* 第一个 event.name 必须是 contact。
* 最后一个 event.name 必须是 release。
* 中间事件 name 可以是：

  * slip
  * left
  * right
  * 或者自定义名字，例如 slip_1, slip_2, left_1, right_2
* 判断发送方式不要只靠 name，要优先靠 modality：

  * modality=vibration：使用 command_label 或 command_id
  * modality=matrix：使用 channel_list 或 payload 配置
* 也就是说，name 是日志标签，modality 决定发送通道。
* 但为了兼容旧接口，contact/release/slip/left/right 这些 name 仍然保留。

目标 3：全局 config 支持统一控制 haptic duration 和 gap 范围

新增 config：

haptic_defaults:
vibration_duration_ms: [150, 300]
matrix_duration_ms: [500, 1000]
inter_event_gap_ms: [300, 1000]
contact_onset_delay_ms: [500, 2000]
release_duration_ms: [150, 300]

语义：

1. 每个 event 可以单独写 duration_ms 或 duration_ms_range。
2. 如果 event 写了 duration_ms，则使用固定 duration。
3. 如果 event 写了 duration_ms_range，则从范围内随机抽取 duration。
4. 如果 event 没写 duration，则按 modality 使用全局默认：

   * contact/release 用 release_duration_ms 或 vibration_duration_ms
   * slip / vibration event 用 vibration_duration_ms
   * left/right / matrix event 用 matrix_duration_ms
5. 每个 event 可以单独写 onset_gap_after_previous_ms。
6. 如果 event 没写 onset_gap_after_previous_ms，则使用 haptic_defaults.inter_event_gap_ms。
7. contact 的 onset delay 如果 event 没写 onset_delay_ms，则使用 haptic_defaults.contact_onset_delay_ms。
8. 所有实际抽到的 duration 和 gap 必须写入 haptic_events.csv。

CSV 字段增加：

* sampled_duration_ms
* global_default_used: true/false
* end_reason
* haptic_episode_completed: true/false
