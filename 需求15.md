现在在 1-back 仓库中新增 wrist0 左右旋转标定功能。

目标：
从 MANUS raw combined JSON 中读取 wrist0 / node_id=0 的 rotation，做一个相对手腕左右旋转标定。不要使用 Vive Tracker，不要依赖全局位置坐标。只使用 MANUS skeleton node0 的 rotation quaternion。

背景：
raw_frames.jsonl 里 skeleton nodes 中 node id 0 有 rotation 字段。node0 position 可能接近 [0,0,0]，不要用 position 判断左右旋转。rotation 是 quaternion，需要用相对标定而不是直接把某个 Euler 角当成绝对左右旋转。

重要限制：

* 不要修改 exp2。
* 不要修改 manus_vive_com。
* 不要修改 MANUS TCP 8888 接口。
* 不要破坏现有 pinch/haptic/1-back 流程。
* 不要强制要求 Vive Tracker。
* 不要假设 wrist0 的绝对 yaw/roll/pitch 就是左右旋转。
* 不要假设 quaternion 顺序，先在代码里集中处理并允许 config 指定。

实现内容：

1. 从 MANUS frame 中读取 wrist0 rotation

新增函数：

extract_wrist_quaternion(device_frame, node_id=0) -> tuple[float, float, float, float] | None

要求：

* 从 device_frame.hand.nodes 或等价结构中找到 node_id=0。
* 读取 rotation quaternion。
* 如果不存在 rotation，返回 None。
* 不要用 tracker。
* 不要用 node position。

2. quaternion 工具函数

实现：

* normalize_quaternion(q)
* invert_quaternion(q)
* multiply_quaternion(q1, q2)
* relative_quaternion(q_current, q_neutral)
* quaternion_to_vector_part(q)
* quaternion_angle(q)

注意：
quaternion_order 来自 config：

* wxyz: [w, x, y, z]
* xyzw: [x, y, z, w]
  内部统一转换为 wxyz。

3. 标定流程

新增 WristRotationCalibrationResult：

neutral_mean_q
left_mean_q
right_mean_q
left_relative_q
right_relative_q
rotation_axis_vector
left_score_mean
right_score_mean
calibration_passed
failure_reason
valid_frame_counts

标定步骤：

1. 提示用户保持 neutral wrist，采集 calibration_duration_s。

2. 提示用户向左旋转手腕，采集 calibration_duration_s。

3. 提示用户向右旋转手腕，采集 calibration_duration_s。

4. 每段取 quaternion 平均。最小实现可以用 normalize 后的逐分量平均，并处理 q 和 -q 同向问题：

   * 如果 dot(q, reference_q) < 0，则 q = -q
   * 然后求均值并 normalize

5. 计算：
   left_relative_q = left_mean_q * inverse(neutral_mean_q)
   right_relative_q = right_mean_q * inverse(neutral_mean_q)

6. 用 left/right 的 relative quaternion vector part 构造一个分类轴：
   axis = normalize(vector(left_relative_q) - vector(right_relative_q))

7. 对当前帧：
   rel_q = current_q * inverse(neutral_mean_q)
   feature = dot(vector(rel_q), axis)

8. 标定阶段得到：
   left_score_mean
   right_score_mean
   threshold = (left_score_mean + right_score_mean) / 2

9. 实时分类

新增函数：

classify_wrist_rotation(current_q, calibration_result) -> WristRotationSample

输出字段：

* monotonic_ms
* node_id
* wrist_rotation_valid
* wrist_rotation_score
* wrist_rotation_class: neutral | left | right | unknown
* distance_to_left
* distance_to_right
* note

分类逻辑：

* 如果 score 明显偏向 left_score_mean，class=left。
* 如果 score 明显偏向 right_score_mean，class=right。
* 如果离 threshold 太近，class=neutral 或 unknown。
* classification_margin 从 config 读取。

6. 集成方式

第一版不要把 wrist rotation 作为 haptic trigger。
只做读取、标定、分类、记录。

在 run_pinch_haptic_1back.py 中：

* 如果 wrist_rotation.enabled=false，完全不影响现有流程。
* 如果 wrist_rotation.enabled=true，在 pinch calibration 之前或之后增加三段 wrist calibration。
* 正式阶段每帧有新 MANUS sample 时，同时记录 wrist_rotation_timeseries.csv。
* summary.json 增加：
  wrist_rotation_enabled
  wrist_rotation_calibration_passed
  wrist_rotation_timeseries_csv

7. 测试

新增 tests/test_wrist_rotation.py：

测试内容：

1. quaternion normalize 正常。

2. q 和 -q 平均时能对齐符号。

3. relative_quaternion(q_neutral, q_neutral) 接近 identity。

4. 用 synthetic neutral/left/right quaternion 能完成 calibration。

5. classify_wrist_rotation 能把 synthetic left/right 分开。

6. quaternion_order=wxyz 和 xyzw 都能正确转换。

7. 缺少 node0 rotation 时返回 invalid sample，不崩溃。

8. wrist_rotation.enabled=false 时不影响现有 1-back/haptic 流程。
