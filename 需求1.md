标定流程：
正式实验前做一次 pinch distance 标定。

流程：

1. 提示被试张开手，采集 open hand 一段时间，得到 max_distance。
2. 提示被试捏合，采集 pinch hand 一段时间，得到 min_distance。
3. 根据 config 计算阈值 a。

a 的公式：
a = min_distance + threshold_ratio * (max_distance - min_distance)

例如 threshold_ratio=0.65 时：
a = 65% * (max-min) + min

threshold_ratio 必须写在 config 里。
不要写死在 Python 中。

haptic 逻辑：
haptic 根据 pinch distance 分两个阶段：

阶段 1：
当 pinch_distance 进入 [a, max_distance]，触发 contact。
contact 是 vibration ESP32 的信号 1。
对应 exp2 里的 vibration command 应该是 contact_enter。

阶段 2：
当 pinch_distance 进入 [min_distance, a]，按 haptic trial plan config 输出后续 haptic 信号。
这些信号可以包括：

* slip：vibration ESP32，信号 3，参考 exp2 slip_start。
* left：matrix ESP32，根据 plan config 中的 channel_list 或 matrix payload 配置。
* right：matrix ESP32，根据 plan config 中的 channel_list 或 matrix payload 配置。
* release：vibration ESP32，信号 2，参考 exp2 contact_exit。

重要规则：

* contact 永远是一个 haptic trial 的第一个信号。
* release 永远是一个 haptic trial 的最后一个信号。
* slip、left、right 是中间可配置信号。
* haptic trial plan config 必须做合法性检查：

  * 第一个事件必须是 contact。
  * 最后一个事件必须是 release。
  * 中间事件只允许 slip、left、right，或者以后明确扩展的合法 event。
  * 如果最后一个不是 release，直接报错，不要静默修正。
  * 如果第一个不是 contact，直接报错。
* 用户以后会自己写很多 haptic trial plan config，用于伪随机、拉丁方平衡等实验设计。
* 程序只需要读取一个 plan config 并执行；不需要自动生成完整拉丁方。

haptic trial plan config 格式示例：

plan_id: plan_001
description: contact first, then slip-left-right counterbalanced example, release last
random_seed: 123

events:

* name: contact
  modality: vibration
  command_label: contact_enter
  command_id: 1
  duration_ms: 150
  trigger_zone: open_zone
  onset_policy:
  type: when_enter_zone
  zone: open_zone

* name: slip
  modality: vibration
  command_label: slip_start
  command_id: 3
  duration_ms: 800
  trigger_zone: closed_zone
  onset_policy:
  type: after_zone_transition
  from_zone: open_zone
  to_zone: closed_zone

* name: left
  modality: matrix
  channel_list: [1, 2, 3]
  duration_ms: 800
  trigger_zone: closed_zone
  onset_policy:
  type: after_previous
  gap_ms: 100

* name: right
  modality: matrix
  channel_list: [5, 6, 7]
  duration_ms: 800
  trigger_zone: closed_zone
  onset_policy:
  type: after_previous
  gap_ms: 100

* name: release
  modality: vibration
  command_label: contact_exit
  command_id: 2
  duration_ms: 150
  trigger_zone: closed_zone
  onset_policy:
  type: after_previous
  gap_ms: 100

zones:
open_zone:
lower: auto_a
upper: auto_max

closed_zone:
lower: auto_min
upper: auto_a

说明：

* open_zone 对应 [a, max_distance]。
* closed_zone 对应 [min_distance, a]。
* contact 在 open_zone 触发。
* 进入 closed_zone 后，执行 plan 中 contact 之后的事件。
* release 必须是最后一个事件。
* left/right 的 channel_list 必须来自 config，不要写死。
* 如果 exp2 matrix 协议需要的不是简单 channel_list，而是特定 payload，请先阅读 matrix_haptic_protocol.py 和 haptic_runtime.py，再把 plan schema 调整成和协议一致的字段。
* 如果无法确认 matrix payload 格式，不要猜，停下来报告。

状态机建议：
可以实现一个简单 haptic trial state machine：

WAIT_OPEN_ZONE:
等待 pinch_distance 进入 [a, max]。
进入后触发 contact。

WAIT_CLOSED_ZONE:
contact 完成后等待 pinch_distance 进入 [min, a]。

RUN_PLAN_EVENTS:
进入 [min, a] 后，按 haptic plan config 执行 slip/left/right/.../release。

REFRACTORY:
一个 haptic trial 完成后，等待 refractory_ms，避免短时间重复触发。

状态切换和所有 event 都要记录 timestamp。
