# 1-Back 任务按键问题解决方案

## 问题描述
- 汉字显示方框 ✅ **已解决** (字体加载成功)
- F/J键无法检测 ❌ **待解决** (但空格和ESC键可以检测)

## 问题分析
根据测试结果，pygame可以检测到按键事件（空格和ESC有效），但F/J键没有被检测到。可能原因：

1. **键盘布局问题** - 可能使用非QWERTY键盘（如AZERTY、DVORAK）
2. **键码映射问题** - F/J键的键码不是102/106
3. **特殊功能键** - F键可能被Fn键或多媒体功能覆盖
4. **系统拦截** - 某些软件可能拦截F/J键

## 解决方案

### 步骤1：诊断问题（先做这个）

运行原始键码测试：
```bash
python test_raw_codes.py
```

**操作：**
1. 点击窗口获取焦点
2. 按 **F** 键
3. 按 **J** 键
4. 按 **ESC** 键退出

**观察输出：**
- 按F键时显示的键码和键名是什么？
- 按J键时显示的键码和键名是什么？

**示例正常输出：**
```
按键检测:
  键码: 102
  键名: 'f'
  unicode: 'f'
  → 这是F键!
```

### 步骤2：尝试不同按键

如果F/J键确实检测不到，尝试：

1. **左右箭头键** - 修改 `config.py`：
   ```python
   KEY_SAME = 'left'      # 左箭头
   KEY_DIFFERENT = 'right' # 右箭头
   ```

2. **A/D键** - 修改 `config.py`：
   ```python
   KEY_SAME = 'a'         # A键
   KEY_DIFFERENT = 'd'    # D键
   ```

3. **数字键** - 修改 `config.py`：
   ```python
   KEY_SAME = '1'         # 数字1
   KEY_DIFFERENT = '2'    # 数字2
   ```

### 步骤3：使用智能版程序（推荐）

运行智能按键版，程序会自动检测可用的按键：
```bash
python nback_task_smart.py
```

**操作流程：**
1. 程序会提示你按两个不同的键
2. 第一个键用于"相同"
3. 第二个键用于"不同"
4. 按ESC键使用默认设置

### 步骤4：详细调试（如果需要）

运行详细调试工具：
```bash
python debug_keys_raw.py
```

这个工具会显示所有按键的原始数据，帮助你找到可用的按键。

## 备选方案

如果所有按键都检测不到（除了空格和ESC），可能是：

1. **键盘驱动程序问题** - 更新键盘驱动
2. **防病毒软件拦截** - 暂时禁用安全软件
3. **系统权限问题** - 以管理员身份运行
4. **远程桌面问题** - 如果是远程连接，尝试本地运行

## 紧急解决方案

如果急需进行实验，可以使用：

1. **鼠标点击替代** - 修改程序使用鼠标点击（左键=相同，右键=不同）
2. **语音输入替代** - 使用其他输入方式
3. **外接键盘** - 尝试使用不同的键盘
