现在不要接真实 TCP。当前 disabled 双任务输出显示 release 结束 session 的逻辑已经生效，但 haptic 时序有严重问题。

现象：
haptic_events.csv 中 contact 正常触发，但 slip、left、right、release 四个事件具有相同 monotonic_ms：

contact: 774587953
slip:    774594593
left:    774594593
right:   774594593
release: 774594593

而它们的 original_planned_onset_ms 分别是：
slip:    774588670
left:    774589859
right:   774591376
release: 774592748

这说明 scheduler 在等待 closed_zone 时，把后续 plan event 的 planned onset 提前算好了。等真正进入 closed_zone 时，这些 planned onset 已经过期，于是 scheduler 在同一个 tick 中把 slip/left/right/release 全部补发了。

这在 disabled logging 下只是日志异常，但如果接真实 ESP32 TCP，会导致多个触觉命令几乎同时发送，持续时间和间隔都失效。



修复目标：
1. contact 仍然在 open_zone 中随机 delay 后触发。
2. contact 触发后进入 WAIT_CLOSED_ZONE。
3. 不要在 WAIT_CLOSED_ZONE 阶段提前计算 slip/left/right/release 的 planned onset。
4. 只有真正进入 closed_zone 的那一刻，才开始调度 contact 后的第一个 plan event。
5. 第一个 closed-zone event 的 planned onset 应该是：
   closed_zone_enter_ms + sampled_gap_after_previous_ms
   或者 current_now_ms + sampled_gap_after_previous_ms
   不能用 contact_end_ms 作为过去的基准。
6. 后续每个 event 的 planned onset 应该基于“前一个 event 的实际 emit time + 实际 duration + sampled gap”，而不是基于早就过期的 planned time。
7. scheduler.update() 每次调用最多返回/emit 一个 ScheduledHapticEvent。
8. 如果当前 now_ms 已经超过 planned onset 很多，也只能 emit 当前这一个 event，然后下一个 event 重新按当前实际 emit/end 时间调度，不能在同一个 tick 连续追赶补发多个事件。
9. haptic_events.csv 中 slip/left/right/release 的 monotonic_ms 必须体现真实间隔，不应完全相同。
10. 如果 event 被 delay 或因 closed_zone 等待导致 planned onset 被重置，要记录 note 或 timing_note，例如：
    planned_after_closed_zone_enter
    planned_after_previous_actual_emit

建议修改 haptic_trial_scheduler.py：

状态机可以改成：

WAIT_OPEN_ZONE:
  等待 open_zone。

PENDING_CONTACT:
  进入 open_zone 后抽 contact delay。
  到点且仍在 open_zone，emit contact。
  contact emit 后进入 WAIT_CLOSED_ZONE。
  不要调度后续事件。

WAIT_CLOSED_ZONE:
  等待 closed_zone。
  一旦进入 closed_zone：
    set closed_zone_enter_ms = now_ms
    set next_plan_event_index = 1
    schedule_next_plan_event(base_ms=now_ms)
    进入 PENDING_PLAN_EVENT
  本次 update 不要马上把所有事件都 emit 完。

PENDING_PLAN_EVENT:
  如果 now_ms < next_event_planned_onset_ms：
    return []
  如果 now_ms >= next_event_planned_onset_ms：
    emit 当前 event
    actual_emit_ms = now_ms
    actual_duration_ms = sampled_duration_ms 或 duration_ms
    previous_actual_end_ms = actual_emit_ms + actual_duration_ms
    if 当前 event 是 release：
       完成 episode，进入 COMPLETED 或 REFRACTORY，根据 allow_multiple_haptic_trials
    else:
       schedule_next_plan_event(base_ms=previous_actual_end_ms)
    return [current_event]

schedule_next_plan_event(base_ms):
  sampled_gap = sample event.onset_gap_after_previous_ms or global/default gap
  planned_onset = base_ms + sampled_gap
  如果 digit onset guard 调整，则 adjusted_onset = ...
  保存 planned/adjusted onset。
  不要循环 emit。
